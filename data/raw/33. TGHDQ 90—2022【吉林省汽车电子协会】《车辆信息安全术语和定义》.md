近年来，随着我国智能网联汽车的高速发展，汽车的信息安全问题日益突出，各个国家、政府和组织都加强了汽车领域的信息安全监管力度。与传统互联网信息安全不同，智能网联汽车的信息安全技术起步较晚，技术种类多样，应用场景复杂，还没有形成统一的汽车信息安全术语和定义，给研究人员之间的技术交流造成了不便。

因此，有必要编制一个通用和规范的汽车信息安全术语和定义。本文件主要从通用类、密码技术类、系统应用类、功能应用类和应用软件类5个类别对汽车信息安全领域的术语进行定义，适用于编写有关汽车信息安全的各类标准和技术文献。

# 4密码技术类术语

# 密钥管理keymanagement

# 密钥协商keyagreement

两个或多个实体通过相互传送一些消息来共同建立一个共享的秘密密钥的协议，且各个实体无法预先确定这个秘密密钥的值。

4.5密文cipher text：加密后的数据。[来源：GM/Z0001-2013，2.60]

# 明文plaintext

4.7

# 加密encryption

4.8

# 解密decryption

加密过程对应的逆过程。[来源：GM/Z0001-2013，2.44]

密码杂凑函数hashfunction

又称杂凑算法、密码散列算法或哈希算法。该算法将一个任意长的比特串映射到一个固定长的比特串，且满足下列三个特性：

a）为一个给定的输出找出能映射到该输出的一个输入是计算上困难的；b）为一个给定的输入找出能映射到同一个输出的另一个输入是计算上困难的；c）要发现不同的输入映射到同一输出是计算上困难的。[来源：GM/Z0001-2013，2.58]

国密算法nationalcipheralgorithm

国密算法是指国家密码局认定的商用加密算法。

商用密码commercialcryptography

由国家密码管理机构指定单位研究和生产的，对不涉及国家秘密内容的信息进行加密保护或者安全认证所使用的密码技术和密码产品。

对称密码算法symmetriccryptographicalgorithm

加密和解密使用相同密钥的密码算法。

[来源：GM/Z0001-2013，2.19]

对称密钥symmetric key

用于对称密码算法的密钥。

[来源：GM/Z0001-2013，2.20]

加密和解密使用不同的密钥的密码算法。其中一个密钥（公钥）可以公开，另一个密钥（私钥）必须保密，且由公钥求解私钥是计算不可行的。

4.15

私钥privatekey

非对称密码算法中只能由拥有者使用的不公开密钥。

[来源：GM/Z0001-2013，2.116]

4.16公钥信息public key information：专用于某个实体的信息，至少包含实体可区分标识符和公钥，还可包括认证机构、实体、公钥、密钥应用约束、有效期或相关算法等相关内容。

数字证书digitalcertificate

关于实体的一种数据，由认证机构的私钥对使用者的公钥进行数字签名后的数据，并无法伪造。

数字签名digitalsignature

签名者使用私钥对待签名数据的杂凑值做密码运算得到的结果，该结果只能用签名者的公钥进行验证，用于确认待签名数据的完整性、签名者身份的真实性和签名行为的抗抵赖性。

[来源：GM/Z 0001-2013，2.113]

证书认证机构certificate authority（CA)

对数字证书进行全生命周期管理的实体。也称为电子认证服务机构，

[来源：GM/Z0001-2013，2.145]

4.20

哈希值hashvalue

散列/杂凌函数的输出值。

4. 21消息鉴别码message authentication code（MAC)又称消息认证码，是消息鉴别算法的输出。[来源：GM/Z0001-2013，2.133]

# 令牌token

由与特定的通信相关的数据字段构成的消息，它包含使用密码技术进行变换的信息。
