# 引 言

随着汽车电动化、网联化、智能化和共享化的发展，汽车的各种新功能、新应用大幅增加，伴随而来的是各种安全隐患，其中信息安全已成为智能网联汽车面临的主要安全风险之一。智能网联汽车频遭恶意攻击，网络信息安全问题不容乐观，已然威胁到车辆用户的人身和财产安全，并对汽车企业造成较大的负面影响。本文件的主要目标是，提出智能网联汽车远程监控系统信息安全技术要求，提升智能网联汽车产品的网络安全水平，避免因网络攻击导致的非法控制、隐私泄露、财产损失甚至人员伤亡，保障智能网联汽车产业健康发展。

# 智能网联汽车远程监控系统信息安全技术要求

# 1范围：本文件规定了智能网联汽车远程监控系统信息安全技术要求，主要包括车载终端与远程监控系统安全通信、远程监控系统与公共平台安全通信、远程监控系统平台安全要求。本文件适用于智能网联汽车远程监控系统。

# 智能网联汽车远程监控系统总体架构

智能网联汽车远程监控系统为车企提供终端管理、信号解析、告警监控、车辆数据统计、电池健康状态评估、车辆预测性维护等功能，满足车企监控和国家监管法规的要求。

车载终端通过电信网络连接远程监控系统，实时上报车况及告警信息。通过远程监控系统可对车载终端下发参数配置等指令。

远程监控系统上报实时车况及告警信息到地方及国家公共平台。

5 车载终端与平台通信安全要求

# 5.1车辆登入安全：车辆登入应符合以下要求：a) 车载终端连接平台时进行双向身份鉴别；b） 平台对终端上传的车辆信息进行校验，避免非法的车辆登入；c）平台记录校验失败的登入信息。

# 5.2安全通信协议：安全通信协议应符合以下要求：a) 身份认证过程需采用非对称加密算法；b）使用TLS1.2同等安全强度的协议或以上版本；c） 使用SM2、密钥长度不低于2048位的RSA或同级别以及更高级的密码算法；d) 具有对应的证书更新及撤销机制：e）证书的有效期不超过10年。

# 5.3数据传输安全：数据传输安全应符合以下要求：a）车载终端向平台上报信息时，应进行加密处理；b）数据加密方式应采用SM4、密钥长度不低于128位的AES或其他同级别以及更高级的密码算法；c）平台需要对数据进行完整性校验；d）链路和设备节点实行余建设。

# 6平台间通信安全要求

# 6.1平台登入安全

登入公共平台应符合GB/T32960.3-2016的要求。

# 6.2安全通信协议

安全通信协议应符合GB/T32960.3-2016的要求。

# 6.3数据传输安全

数据传输安全应符合GB/T32960.3-2016的要求。

# 6.4安全审计：安全审计应符合以下要求：a）审计内容应包括平台连接、平台登入、登入异常、数据校验异常等重要的安全相关事件；b）审计记录应包括日期、时间、类型、主体标识和结果等；c）保护审计记录，有效期内避免受到非授权的访问、篡改、覆盖或删除等；d）应支持按公共平台提供与其相关的审计信息及审计分析报告。

# 平台安全要求

# 7.1一般要求

应满足GB/T22239-2019安全等级保护第三级及以上的要求。

# 7.2身份鉴别：身份鉴别应符合以下要求：a）对用户进行身份标识和鉴别；b）采用口令、密码技术、生物技术等两种或两种以上组合的鉴别技术对用户进行身份鉴别，且其中一种鉴别技术至少应使用密码技术来实现；c）采取加密存储保护身份鉴别信息安全；d）启用登录失败处理功能，可采取结束会话、自动退出和限制非法登录不超过5次等措施。

# 7.3访问控制,访问控制应符合以下要求：a）对用户分配不同的角色和权限；b）用户管理进行权限分离，仅该角色用户所需的最小权限；c）限制用户的访问权限，按实际业务需求控制用户对应用的访问；d）严格控制终端参数设置功能的访问授权。

# 7.4多租户安全

如果采用多租户设计，应符合如下要求：

a）应用体系结构和数据模型的设计应确保止确的数据隔离：b）通过加密保障数据安全，防止由于应用程序漏洞或者恶意特权用户泄漏敏感信息；c）保存所有数据访问记录，并定期审计。

# 7.5车载终端管理

车载终端管理应符合如下要求：

a）保证车载终端密钥初始化过程的保密性、完整性：b）保证车载终端密钥更换过程的保密性、完整性；c）定期更换密钥；d）车载终端及时升级到最新版本。

# 7.6终端安全事件监管

终端安全事件监管应符合如下要求：

a）应对车载终端上报的信息安全事件日志进行监控管理：b）对车载终端产生的信息安全问题及时进行告警通知；c）在车载终端产生信息安全问题后，能为信息安全应急响应提供车载终端相关数据以及追溯手段。

# 7.7车辆安全监控

车辆安全监控应符合如下要求：

a）具备车辆故障监控和安全报警的功能：b）及时对故障和报警进行处置并上报公共平台。

# 7.8平台数据安全

平台数据安全应符合如下要求：

a）对敏感数据加密存储；b）设置数据访问规则，非授权用户不能访问和更改数据；c）远程监控系统信息存储应保证完整性；d）数据在线存储时间不少于6个月；(e 建立数据备份机制，每月对数据进行全量备份，每周对数据进行增量备份，数据恢复时间不超过3小时。
