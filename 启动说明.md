# 智能问答 Agent 启动说明

## 方式一：本机运行（开发调试）

### 1. 启动后端 API

```bash
cd ~/Agent/react_agent+rag
uvicorn main_fastapi:app --host 0.0.0.0 --port 8000
```

后端启动后访问 `http://localhost:8000/docs` 查看 Swagger API 文档。

### 2. 启动前端页面

```bash
cd ~/Agent/react_agent+rag
streamlit run streamlit_app.py
```

前端启动后访问 `http://localhost:8501` 使用图形界面问答。

---

## 方式二：Docker 运行

### 1. 构建镜像

```bash
cd ~/Agent/react_agent+rag
docker build -t react-agent-rag:latest .
```

> 首次或代码修改后需要重新构建。

### 2. 启动容器   已经有的 直接启动 docker start rag-api

```bash
docker run -d --name rag-api -p 8000:8000 \
  -v /home/lyj/sentence-transformers/BAAI/bge-small-zh-v1.5:/app/models/bge-small-zh-v1.5 \
  -v /home/lyj/sentence-transformers/BAAI/bge-reranker-v2-m3:/app/models/bge-reranker-v2-m3 \
  -e EMBEDDING_MODEL_PATH=/app/models/bge-small-zh-v1.5 \
  -e RERANK_MODEL_PATH=/app/models/bge-reranker-v2-m3 \
  react-agent-rag:latest
```

### 3. 启动前端

Docker 只运行后端 API，前端仍需在本机运行：

```bash
cd ~/Agent/react_agent+rag
streamlit run streamlit_app.py
```

## 常用命令

### Docker 容器管理

| 操作 | 命令 |
|---|---|
| 查看日志 | `docker logs rag-api` |
| 实时追踪日志 | `docker logs -f rag-api` |
| 停止容器 | `docker stop rag-api` |
| 启动已有容器 | `docker start rag-api` |
| 重启容器 | `docker restart rag-api` |
| 删除容器 | `docker rm -f rag-api` |

### API 测试

```bash
# 普通问答
curl -X POST http://localhost:8000/api/ask \
  -H "Content-Type: application/json" \
  -d '{"question": "你好"}'

# 流式问答
curl -X POST http://localhost:8000/api/ask_stream \
  -H "Content-Type: application/json" \
  -d '{"question": "你好"}'
```

## 路径说明

| 路径 | 说明 |
|---|---|
| `http://localhost:8000` | 后端 API（无前端页面，访问显示 404 属正常） |
| `http://localhost:8000/docs` | Swagger API 文档 |
| `http://localhost:8501` | Streamlit 前端页面 |
